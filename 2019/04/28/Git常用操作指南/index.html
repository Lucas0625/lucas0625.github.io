Git 常用操作指南

.gitignore规范

  • 所有空行或者以注释符号 ＃ 开头的行都会被 Git 忽略。
  • 可以使用标准的 glob 模式匹配。
  • 匹配模式最后跟反斜杠（/）说明要忽略的是目录。
  • 要忽略指定模式以外的文件或目录，可以在模式前加上惊叹号（!）取反。
    [注]:所谓的 glob 模式是指 shell 所使用的简化了的正则表达式。
    1. 星号（*）匹配零个或多个任意字符
    2. [abc] 匹配任何一个列在方括号中的字符（这个例子要么匹配一个 a，要么匹配一个 b，要么匹配一个 c）
    3. 问号（?）只匹配一个任意字符
    4. 如果在方括号中使用短划线分隔两个字符，表示所有在这两个字符范围内的都可以匹配（比如 [0-9] 表示匹配所有 0 到 9 的数字）。

example:

查看文件之间的差别

  1. git diff file 查看工作目录中当前文件和暂存区域快照之间的差异
  2. git diff –cached / –staged 查看已经暂存起来的文件和上次提交时的快照之间的差异

提交文件

  1. git commit -m ‘up’ 将暂存区的文件提交
  2. git commit -a -m ‘up’ 跳过add步骤，将工作区的文件直接提交

删除文件

  1. git rm file 从暂存区删除，并连带从工作目录中删除指定的文件
  2. git rm -f file 如何删除之前修改过，并且已加入暂存区，则必须强制删除
  3. git rm –cached file 将文件从git仓库中删除，但是仍保留在工作目录中

修改文件名

  1. git mv file_form file_to

查看提交历史

  1. git log 按照提交历史列出所有更新，最近的更新排在最上面
  2. git log -p -n 显示每次提交的内容差异，显示最近的n次提交
  3. git log –stat 显示每次提交的简要增改行统计
  4. git log –pretty=oneline 将每个提交放在一行显示

  5. git log –pretty=format:”%h - %an, %ar : %s” 定制要显示的记录格式

  6. git log –pretty=format:”%h %s” –graph 增加ASCII字符串表示的简单图形，展示每个提交所在的分支及其分化衍合情况

  7. 其他选项

  8. git log –since=2.weeks 显示最近两周的提交

撤销操作

  1. git commit –amend 撤销刚才的提交操作，使用当前的暂存区域快照提交，如果刚才提交完没有作任何改动，相当于有机会重新编辑提交说明
  2. git reset HEAD file 取消已暂存的文件
  3. git checkout – file 取消工作区对文件的修改!!!!!!!!!!(此过程不可逆)

管理远程仓库

  1. git remote 列出远程仓库的名字
  2. git remote -v 显示对应的克隆地址
  3. git pull 拉取远程仓库的更新
  4. git push 推送数据到远程仓库

分支操作

  1. git branch testing 新建testing分支
  2. git branch 显示当前分支
  3. git branch -a 显示所有分支
  4. git checkout testing 转换到testing分支
  5. git checkout -b testing 新建testing分支，并转向testing分支
  6. git branch -d testing 删除testing分支

  7. git merge testing 合并testing分支到当前分支

  8. git branch -v 查看各个分支最后一个提交的对象的信息

  9. git branch –merged 筛选出与当前分支合并的分支，一般来说列表中没有*的分支通常都可以删除
  10. git branch –no-merged 筛选出尚未与当前分支合并的分支
  11. git rebase master 把当前分支里提交的改变移动到master分支里
  12. git rebase –onto master server client 取出 client 分支，找出 client 分支和 server 分支的共同祖先之后的变化，然后把它在 master 上重演一遍
  13. git rebase master server 取出特性分支 server，然后在主分支 master 上重演

储藏(stashing)操作

“‘储藏”“可以获取你工作目录的中间状态——也就是你修改过的被追踪的文件和暂存的变更——并将它保存到一个未完结变更的堆栈中，随时可以重新应用。

  1. git stash 往堆栈推送一个新的储藏，
  2. git stash list 查看现有的储藏
  3. git stash apply 恢复最近一次储藏，但是不从堆栈中删除该储藏
  4. git stash apply stash@{2} 恢复指定储藏，但是不从堆栈中删除该储藏
  5. git stash pop 恢复最新一次储藏，并从堆栈中删除它
  6. git stash drop stash@{2} 删除指定储藏
